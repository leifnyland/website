Robots - Github

This was my first forray into adversarial learning. My neural network played a 2 player game with multiple robots per team, with the goal of having the most robots left by the end of 100 rounds of play.

Objective

Adversarial Learning allows a DNN to learn both about concepts of a game, and how to win. As a programmer, it is advantageous because I do not need to know how to win, or even how to play. All I need to do is have a rough idea of what might be important, and then know how to rate the robot on playing well or poorly. This project is my first look into adversarial learning and the power its use of problem solving and artificial intellegence.

Methodology

The robots game came from a class where students programmmed various bots by hand. Later, this same was adapted to run on a DNN. The game is played on various grid maps with different obstacle arrangements. The rules are as follows:

  • Each game lasts 100 rounds
  • Every 10 rounds, starting with round 0, 5 robots from each team spawn
  • The robots have the following stats: 50 health, attacks do 5 damage, self destruct does 15 damage to all 4 adjacent squares, and guarded robots take half damage.
  • At the beginning of every round, each robot does one of 10 moves: move (up, down, left, right), attack (up, down, left, right), guard, or self destruct.
  • All robots on the same team run the control code submitted for that player. Each robot can see the game state in some manner, which is controlled by the programmer.
  • There are other rules to control edge cases, which I will not go into here.

For adversarial learning, the input can be modified to include all sorts of input parameters: current health, location of opponents/teammates, heath of opponents/teammates, round, and pretty much anything else immagineable. A big part of this project was to decide which parameters to include in the input array and the radius of vision, as it all drastically affects learning speed.

Results

This project was part of a final project for a class at Appalachian State University. After the whole class trained a program for the default map and 2 other maps, we held tournaments for each map. Mine won first place in all tournaments, although I feel like that was more of a determiner of understanding more than skill. The class was taught in a very hands on/pproject style, and so the "winner" was usually the person who understood the concepts of the project the best. Most understanding came through self exploration, and I feel like my advantage came from grasping how the robot game state view actually impacted learning speed and robot "knowledge". I think many people, including myself, wasted a lot of time giving the neural network too much information, which increased the size of our DNN and slowed down learning. I would like to do this project again, and see if I can place better in the "zoo" now that I know more. The zoo is a compilation of previous robot control programs, all hardcoded. It allowed us to have a rough idea of performance and improvement, as we could see how many of the "zoo" bots we could beat. No one in the class could reliably beat any of the 3 best zoo bots, which I found dissapointing and is a big reason why I would like to work more on this project. It should be possible to produce a comparible program using a DNN, and it is just my knowledge and implementation holding me back.
